Keep Awake — Changelog
======================

v2.6.0 (2026-05-11)
--------------------
- New: Spanish (Español) language option — full UI translation in Settings > System
  covering all UI strings: tray menu, settings window, dialogs, notifications, day names
- New: weekly usage summary — on first startup each week a tray notification shows
  last week's total keep-awake time, number of active days, and daily average;
  state stored in .last_summary_week so the notification fires once per week only
- GitHub: added MIT LICENSE
- GitHub: added issue templates (bug report, feature request) and PR template with checklist
- GitHub: added GitHub Actions release workflow — on every v* tag push, GitHub automatically
  builds the exe with PyInstaller, compiles the installer with Inno Setup, and publishes the
  release with keep_awake_setup.exe attached (no manual build steps required)
- GitHub: repository topics set (windows, python, system-tray, productivity, keep-awake, etc.)
- Fixed: GitHub Actions workflow — replaced spec file reference with inline PyInstaller flags
  since keep_awake.spec is not tracked in the repository
- Fixed: Español radio button overlapping the separator in Settings > System — separator row
  was not updated when the third language option was added

v2.5.1 (2026-05-11)
--------------------
- Fixed: crash on quit — hotkey thread null check was incorrectly placed; now safely
  checks thread exists and has a valid ident before posting WM_QUIT
- Fixed: _hotkey_thread was never reset to None after unregister, leaving a dead reference
- Fixed: closing the custom interval dialog without input silently reset interval to 60s;
  now closing without entering a value leaves the current interval unchanged
- Fixed: import keep_awake inside monitor loops (schedule, meeting, battery, lock) was
  unhandled — now wrapped in try/except so a startup failure doesn't silently kill monitors
- Fixed: version parser in auto-updater failed on pre-release tags (e.g. v2.5.1-beta);
  now strips suffix before comparing — pre-release releases no longer skipped
- Fixed: _parse_duration accepted negative integers (e.g. "-10"); now returns error
- Fixed: dead import of _settings_to_dict in _import_settings() removed
- Fixed: duplicate _refresh_listbox() call in _activate_profile() removed
- Fixed: b.get("start") check unsafe for empty list — replaced with "start" in b and b["start"]
- Refactored: duplicated schedule migration logic (old schedule_start/end format) extracted
  into _migrate_schedule_blocks() helper in state.py — used in all 3 call sites
- Refactored: Optional[object] type hint for active_since_dt replaced with Optional[datetime.datetime]
- Fixed: _format_duration returned unreachable "or '0s'" — replaced with explicit conditional
- Tests: expanded from 33 to 51 tests
  - Added TestFormatHotkey (5 tests: Ctrl+Shift+K, single modifier, all modifiers, non-alpha VK)
  - Added TestParseVersion (7 tests: semver, v-prefix, pre-release stripping, comparison)
  - Added TestParseDuration edge cases: 1m30s, float strings, negative strings
  - Added TestInScheduleWindow: weekend excluded/included, empty blocks

v2.5.0 (2026-05-11)
--------------------
- New: Windows installer (keep_awake_setup.exe) built with Inno Setup
  - Installs to Program Files, creates Start Menu shortcut
  - Optional desktop icon and "Start with Windows" task during install
  - Clean uninstall via "Add or remove programs"
- New: auto-update checker — app queries GitHub releases API 30s after startup;
  if a newer version is available, shows a tray notification and prompts to download
  and install automatically
- New: VERSION constant in state.py — bumping it triggers auto-update for all users
- New: self-signed code signing certificate (CN=Keep Awake, O=carlomazo, C=BR);
  installer and exe are signed — SmartScreen now shows publisher name instead of "unknown"
- Fixed: autostart registry entry pointed to start.bat instead of the exe when running
  as a compiled binary — now detects frozen mode via sys.frozen and registers correctly

v2.4.2 (2026-05-10)
--------------------
- Fixed: icon.stop() called twice on quit — caused occasional crash on exit
- Fixed: TypeError when auto-stop value is None (missing None guard before comparison)
- Fixed: schedule_blocks reset to [] when all time fields were blank — now preserves existing blocks
- Fixed: duplicate key-capture binding in Settings caused UI to freeze after capturing a hotkey
- Fixed: race condition in tooltip — time.monotonic() subtracted from None when start_time unset
- Fixed: file handle leak — open(log_file).close() replaced with proper `with` block
- Fixed: Export HTML showed no warning when log file did not exist
- Fixed: TypeError on boot when schedule_days is null in settings.json — now defaults to empty set
- Fixed: TypeError in custom auto-stop dialog when input is empty (None guard added)

v2.4.1 (2026-05-10)
--------------------
- Refactored: split monolithic keep_awake.py (~2100 lines) into focused modules
  - state.py   — AppState dataclass, settings load/save, shared singleton
  - core.py    — translations, T(), tooltip, icon generation, ctypes helpers, schedule logic
  - monitors.py — all background loops (keep-awake, tooltip, schedule, meeting, battery, lock)
  - settings_ui.py — full Settings window (Tkinter)
  - keep_awake.py  — orchestrator: main(), tray menu, start/stop, hotkey
- Replaced ~25 global variables with a single AppState dataclass instance (state singleton)
- Added centralized _shutdown threading.Event — all loops exit cleanly on quit
- Added tests/ — 33 unit tests covering tooltip, schedule window, duration parsing/formatting,
  icon color logic, and translation fallback (all passing)

v2.4.0 (2026-05-10)
--------------------
- New: "Start paused" option in Settings > System — app opens without activating keep-awake
- New: "Pause when screen is locked" in Settings > System — resumes only if app was active before locking
- New: active profile name shown in tray Profiles submenu (e.g. "Profiles (Work)")
- Settings reorganized: Keep-awake methods moved from System tab into General tab
  (General = Interval + Auto-stop + Keep methods; System = autostart, detection, theme, hotkey)

v2.3.0 (2026-05-10)
--------------------
- New: configurable keep-awake methods in Settings > General
  - System API: SetThreadExecutionState (prevent sleep / screen off) — on by default
  - Mouse nudge: move cursor exactly 1px and back, returns to original position — on by default
  - Key press: simulate a configurable key press — off by default
    - Key is user-selectable (click button, press any key); default is Scroll Lock
    - Any combination can be active; at least one must remain enabled

v2.1.0 (2026-05-10)

- Log: added Month view — sessions grouped by month with total, days active, and avg/day
- Log: Chart/Week/Month buttons toggle back to List when clicked again
